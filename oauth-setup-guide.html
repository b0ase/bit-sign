bit-sign OAuth Setup Guide

Register 5 OAuth apps. Prod only. Each takes ~3 minutes.

Production URL

https://bit-sign.online

Total env vars needed

11 (5 pairs + 1 optional tenant)

Google Easy

Consoleconsole.cloud.google.com/apis/credentials
Callback URLhttps://bit-sign.online/api/auth/google/callback
Scopesopenid, email, profile
  1. Go to Google Cloud Console → Credentials
  2. Click Create Credentials → OAuth Client ID
  3. If prompted, configure the OAuth Consent Screen first:
    • App name: bit-sign
    • User support email: your email
    • Authorized domain: bit-sign.online
    • Publish status: Production (submit for verification later)
  4. Application type: Web application
  5. Name: bit-sign
  6. Authorized redirect URIs: https://bit-sign.online/api/auth/google/callback
  7. Copy Client ID and Client Secret
GOOGLE_CLIENT_ID=your-client-id GOOGLE_CLIENT_SECRET=your-client-secret
Google requires OAuth consent screen verification for apps with 100+ users. Start in "Testing" mode (100 user cap) then submit for review when ready.

X (Twitter) Medium

Portaldeveloper.x.com/en/portal/dashboard
Callback URLhttps://bit-sign.online/api/auth/twitter/callback
Scopestweet.read, users.read
Auth typeOAuth 2.0 with PKCE
  1. Go to X Developer Portal
  2. Create a Project if you don't have one, then create an App
  3. In App Settings → User authentication settings → Edit
  4. Enable OAuth 2.0
  5. Type of App: Web App
  6. Callback URI: https://bit-sign.online/api/auth/twitter/callback
  7. Website URL: https://bit-sign.online
  8. Copy the Client ID and Client Secret (shown once!)
TWITTER_CLIENT_ID=your-client-id TWITTER_CLIENT_SECRET=your-client-secret
X requires at least the Free tier ($0). Make sure OAuth 2.0 is enabled (not just OAuth 1.0a). The Client Secret is shown only once when you first set up auth.

LinkedIn Medium

Consolelinkedin.com/developers/apps
Callback URLhttps://bit-sign.online/api/auth/linkedin/callback
Scopesopenid, profile, email
ProductSign In with LinkedIn using OpenID Connect
  1. Go to LinkedIn Developer Apps
  2. Click Create App
  3. App name: bit-sign, Company: select your LinkedIn company page (required)
  4. In the Products tab, request access to "Sign In with LinkedIn using OpenID Connect"
  5. In Auth tab, add redirect URL: https://bit-sign.online/api/auth/linkedin/callback
  6. Copy Client ID and Client Secret from Auth tab
LINKEDIN_CLIENT_ID=your-client-id LINKEDIN_CLIENT_SECRET=your-client-secret
LinkedIn requires a Company Page to create an app. The "Sign In with LinkedIn using OpenID Connect" product is auto-approved, but takes a few minutes to activate. If the old "Sign In with LinkedIn" shows, pick the OpenID Connect version.

Discord Easy

Consolediscord.com/developers/applications
Callback URLhttps://bit-sign.online/api/auth/discord/callback
Scopesidentify, email
  1. Go to Discord Developer Portal
  2. Click New Application, name it bit-sign
  3. Go to OAuth2 in the left sidebar
  4. Add redirect: https://bit-sign.online/api/auth/discord/callback
  5. Copy Client ID and Client Secret (click Reset Secret if needed)
DISCORD_CLIENT_ID=your-client-id DISCORD_CLIENT_SECRET=your-client-secret
Discord is the simplest. No approval needed, no limits. Works immediately.

Microsoft Medium

PortalAzure Portal → App Registrations
Callback URLhttps://bit-sign.online/api/auth/microsoft/callback
Scopesopenid, profile, email, User.Read
  1. Go to Azure Portal → App Registrations
  2. Click New registration
  3. Name: bit-sign
  4. Supported account types: Accounts in any organizational directory and personal Microsoft accounts
  5. Redirect URI: Platform = Web, URI = https://bit-sign.online/api/auth/microsoft/callback
  6. Click Register. Copy the Application (client) ID
  7. Go to Certificates & secrets → New client secret → Copy the Value (shown once!)
MICROSOFT_CLIENT_ID=your-application-client-id MICROSOFT_CLIENT_SECRET=your-client-secret-value
Use tenant = common (default) to allow both personal Microsoft accounts and org accounts. If you only want personal accounts, use consumers.

Optional: MICROSOFT_TENANT_ID=common (defaults to common if not set)

After Setup

  1. Add all env vars to .env.local on your machine
  2. Add them to Vercel: Project Settings → Environment Variables (Production)
  3. Redeploy (or push a commit)
  4. Test each provider by clicking the link buttons on your account page
